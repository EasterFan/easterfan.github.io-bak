第二章 java封装

Java 封装

什么是封装

  • 将类的某些信息隐藏在类内部，不允许外部程序直接访问
  • 通过该类提供的方法来实现对隐藏信息的操作和访问
  • 隐藏对象的信息
  • 留出访问的接口
    就像路边的ATM机， 钞票是ATM机的重要信息，它隐藏了钞票的总数和存储方法，不允许外部程序的直接访问，但同时，ATM机又提供了插卡口/取钞口/操作屏等接口，使用户通过简单操作取出钞票。

封装的特点

  • 隐藏类的实例细节，方便修改和实现
  • 只能通过规定的方法访问数据

封装代码

封装三步骤

对类进行封装

public class Cat {
    // 成员属性
    private String name;//将name属性设为私有
    int age;
    double weight;
    String specie;

    // 无参构造方法
    public Cat(){

    }


    // 有参构造方法
    public Cat(String name,int age,double weight,String specie){
        this.name = name;
        this.age = age;
        this.weight = weight;
        this.specie = specie;
    }

    // set方法获得对象传递的方法
    public void setName(String name){
        this.name = name;

    }

    // get方法向对象返回逻辑处理后的结果
    public String getName(){

        return this.name;
    }


}

对象通过get/set方法访问类

public class CatInit {
    public static void main(String args[]){
        //实例化对象
        Cat katty = new Cat();

        // 把Tom这个name属性传到Cat类中
       katty.setName("Tom");

       // 从Cat类中得到返回值
       System.out.print(katty.getName());
    }
}

有参构造方法调用set方法

我们封装了Cat类，在set方法中添加一段逻辑代码，检查输入的参数是否符合逻辑（如将年龄设为负数）；
在有参构造函数中，我们应该在构造函数中调用set方法，即在this.name=name前调用set方法检查参数是否符合逻辑

包进行类管理

包对类的管理，就像文件夹对文件的管理，同一个包中不能有同名的类，同名的类可以在不同的包中。
那么，怎样跨包访问类呢？ 

//定义包
package com.easter.zoo
//1.导入包(包名+类名)
import com.easter.zoo.animal;
//2.导入包（在new时写绝对路径）
com.easter.zoo.animal.Cat tom = new com.easter.zoo.animal.Cat();

static关键字

public class Cat(){

public static price;

}

这就涉及类变量和成员变量的理解了。
在一个类中，用static修饰的变量是类变量，反之是成员变量。
类变量属于类，从这个类中new出来的所有对象共用这一个类变量，类变量的存储区域是被共享的。
就像Tom和Jerry作邻居时，电梯就是一个类变量，Tom和Jerry都可以装修电梯，后装修的覆盖先装修的。

  1. 关于变量的生命周期
    成员变量在相应的对象被销毁后，成员变量被释放
    类变量在相应的类被销毁后，类变量释放（相比类变量生命周期长）

  2. 关于static修饰的范围 static只能修饰类中的成员变量和成员方法，不能修饰类或局部变量。

  3. 关于类变量的访问方法： 

    one.price = 100;
Cat.price = 100;

    既可以通过对象访问，也可以通过类名访问（推荐类名访问）

  4. 关于静态方法和非静态方法之间的访问：
    静态方法中不能访问同类成员属性和方法，成员方法可以访问静态属性。
    如果我们非要在静态方法中访问非静态方法和变量呢？
    可以通过new一个对象，再通过对象去调用非静态方法和属性。 

    public static void eat(){
     //直接调用报错
     eat();

     // 通过new对象调用不报错
     Cat tom = new Cat();
     tom.eat();
 }
